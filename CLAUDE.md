## Project Overview

This is a React admin dashboard built with Vite, TypeScript, and TanStack Router. It uses shadcn/ui components styled with Tailwind CSS v4, and integrates Clerk for authentication. The project follows a feature-based architecture with file-based routing.

## Architecture

### File-based Routing with TanStack Router

Routes are defined in `src/routes/` using TanStack Router's file-based system. The route tree is auto-generated in `src/routeTree.gen.ts` - do not edit this file directly.

- `__root.tsx` - Root route with QueryClient provider, dev tools, and error boundaries
- `_authenticated/` - Layout route for authenticated pages (uses `AuthenticatedLayout` component)
- `(auth)/` - Route group for authentication pages (sign-in, sign-up, etc.)
- `(errors)/` - Route group for error pages (401, 403, 404, 500, 503)

Route path conventions:
- Underscore prefix (`_authenticated`, `_auth`) hides the segment from the URL but creates a layout
- Parentheses (`(auth)`, `(errors)`) create logical groups without affecting the URL

### Directory Structure

```
src/
├── components/
│   ├── data-table/      # Reusable TanStack Table components (pagination, filters, etc.)
│   ├── layout/          # App layout components (sidebar, header, nav)
│   └── ui/              # shadcn/ui components (auto-generated, ignored by eslint/knip)
├── context/             # React context providers (theme, font, layout, search)
├── features/            # Feature-based modules
│   ├── auth/           # Authentication pages and forms
│   ├── dashboard/      # Dashboard overview
│   ├── tasks/          # Tasks management with data tables
│   └── users/          # User management with data tables
├── hooks/               # Custom React hooks
├── lib/                 # Utilities (cookies, error handling, utils)
├── routes/              # TanStack Router file-based routes
├── stores/              # Zustand stores (auth-store)
└── styles/              # Global CSS with Tailwind
```

### State Management

- **Zustand** (`src/stores/`) - Global state like authentication (see `auth-store.ts`)
- **TanStack Query** - Server state management with caching and invalidation
- **React Context** - Theme, font, layout, and search state

### Data Tables

The project uses TanStack Table with URL state synchronization via `useTableUrlState` hook (`src/hooks/use-table-url-state.ts`). This keeps pagination, filtering, and sorting in sync with the URL.

Feature data tables (e.g., Tasks, Users) follow this pattern:
- `*-table.tsx` - Main table component using TanStack Table
- `*-columns.tsx` - Column definitions with filtering/sorting
- `*-dialogs.tsx` - Dialog components for create/edit/delete
- `*-primary-buttons.tsx` - Action buttons
- `*-bulk-actions.tsx` - Bulk row operations
- `*-provider.tsx` - Context provider for table state

### UI Components

The `src/components/ui/` directory contains shadcn/ui components. These are auto-generated and should be treated as read-only. ESLint and knip ignore this directory.

## Key Patterns

### Creating a New Route

1. Create a file in `src/routes/` following TanStack Router conventions
2. For authenticated pages, place under `_authenticated/`
3. Export a `Route` object created via `createFileRoute()`

### Adding a Feature with Data Table

1. Create feature folder under `src/features/`
2. Add `index.tsx` for the main component
3. Create `components/` subdirectory with:
   - `*-table.tsx`, `*-columns.tsx`
   - `*-dialogs.tsx`, `*-primary-buttons.tsx`
   - `*-provider.tsx` for shared state
4. Create route file in `src/routes/_authenticated/` that imports and renders the feature component

### Authentication Flow

- Auth state managed by `useAuthStore` Zustand store
- Access token stored in cookies via `lib/cookies.ts`
- Global query error handler in `main.tsx` handles 401/403/500 errors
- Session expiration triggers automatic redirect to sign-in with return URL
